# Project Context

This project builds an internal reservation system for an educational center.

The application must be:

- simple to maintain
- stable
- understandable by non-specialist staff
- easy to deploy
- low operational complexity


## Technology

- Google Apps Script
- HTML Service
- Google Sheets as initial persistence layer


## Constraints

The application will be maintained by the school's IT coordinator who is not a specialized software developer.

The code must therefore prioritize:

- readability
- simplicity
- minimal operational complexity
- clear structure


## Design priorities

Prefer solutions that:

- reduce maintenance burden
- minimize infrastructure complexity
- remain understandable after long periods without modification
